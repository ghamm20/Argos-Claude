### Doctrine (locked)

The **Seven USB-Native Rules** (`docs/01-SEVEN-RULES.md`) are non-negotiable for v1.0. Every commit on `main` since Phase 1 has passed all 7. Enforcement is automated via `scripts/verify-argos.mjs` (a.k.a. `npm run verify`), wired into `npm run check` and CI.

The **Diagnosis Discipline** (3-axis: model-store integrity / arch-compat / hardware-envelope / persona-logic) is the operator's debugging contract. Never blur axes.

## What's deferred (and to where)

### Power Mode / 5090 branch — when better hardware lands

| Capability | Reason held back | Restoration path |
|---|---|---|
| Bart on `huihui_ai/gpt-oss-abliterated:20b` | 8 tok/s + 39% offload on 8 GB RTX 3060 Ti (Phase 1.5 measured) | Rebind in `lib/personas.ts` + pull model; e4b stays as fallback |
| Juniper on `hf.co/HauhauCS/Qwen3.5-9B-Uncensored-HauhauCS-Aggressive:Q4_K_M` | Architecture (`qwen35`) not supported by bundled Ollama 0.24.0 — see Decision #40 | Wait for upstream Ollama support OR pin a fork; then rebind |
| Sage on `alfaxad/wild-gemma4:e4b` (original) | Operator wiped Ollama; current `e4b:latest` is a different blob | Pull the original alfaxad build + re-bind in personas.ts |
| Bobby on `Jarcgon/gemma-4-abliterated:e2b-v2` | Same wipe; gemma2-2b is the validated stopgap | Same re-pull pattern |

The four `intendedModel` fields in `lib/personas.ts` record the would-be mappings so restoration is a one-line change.

### v1.1 (next minor — operator-prioritized improvements)

- **Voice install bundle.** Auto-detect + offer "install voice" in a Settings tab. Currently operator-manual per `docs/VOICE.md`.
- **VAD (voice activity detection) for hands-free dictation.** AudioWorklet + silence detector; ~80 lines. Currently tap-to-record.
- **Per-persona thinking toggle.** `Persona.exposeThinking` flag for personas that genuinely want reasoning-trace visibility (none currently want it; `think:false` is global in `/api/chat`).
- **Chain append O(log n).** Cache tail hash in memory; only re-read on disk-side change. Current O(n) is fine up to ~100k entries.
- **Real-time voice receipts UI.** Audit chain already records `voice.transcribed` / `voice.spoken`, but no dedicated history surface.
- **Vault re-rank.** Currently pure cosine; v1.1 adds BM25 + a small re-ranker (still local; same `nomic-embed-text` for the first stage).

### v2 (deferred — bigger structural)

- **Memory tiers** (5-tier persistent context) — Tier 1 (short-term scratchpad) is the next minor; Tiers 2-5 (working / episodic / semantic / archival) need persistent storage design beyond current `state/sessions/`.
- **Vector store** (e.g. Chroma, LanceDB, Qdrant) when chunk count exceeds ~50k. Current in-memory cosine is right for v1.0 scale (validated to 19 docs / 76 chunks).
- **Workflow agent** (Tier 10) — multi-step orchestration. The `workflow.executed` audit kind is reserved; no code yet.
- **Proposer** (Tier 8) — draft-and-review diffs, prompts, configs. Audit kinds `proposal.*` reserved.
- **Research subsystem** (Tier 3.5) — web-fetch + FS indexer. Currently network-off by Rule 5.
- **Encryption at rest** (state/sessions, vault, audit chain). Currently filesystem-level only.
- **Multi-workspace switcher.** UI exists as a stub; v1.0 has Operator workspace only.

### Cut (not coming, not v1.x scope)

- Cloud sync / SaaS clone / "offline mode of a cloud product"
- Easter eggs, ring activation, ambient modes
- Tool-calling from inside the chat surface (Tools dock launches external tools; in-chat tool invocation is workflow-agent territory)
- Fake AGI / sentience simulator framing — ARGOS is an operator tool, not a personality

## Acceptance checklist for "is this v1.0?"

A build is v1.0-ready iff ALL boxes pass:

- [x] `npm run lint` clean
- [x] `npm run typecheck` clean
- [x] `npm run verify` — Seven Rules 7/7 PASS
- [x] `npm run build` clean (Next.js production build)
- [x] `npm run audit:verify` PASS (empty chain or valid chain)
- [x] `npm run voice:smoke-offline` 12/12 PASS (scaffold integrity)
- [x] `scripts/validate-e4b.mjs` PASS (when on a rig with the models installed)
- [x] All four personas declared with explicit status (`live` / `selectable` / `not_configured`)
- [x] Bart is the default boot persona
- [x] Bart's v1.0 system prompt is preserved verbatim
- [x] `nomic-embed-text` is in Ollama (vault retrieval functional)
- [x] Both launcher payloads (Desktop NTFS + H: FAT32) carry the same `.next` build
- [x] All deployed payload reports exist (`PHASE_1_REPORT.md` through `PHASE_6_REPORT.md`)
- [x] No `console.log` / `any` / empty catch blocks in `app/` / `components/` / `lib/`
- [x] CHANGELOG has the v1.0 section (this commit)

All boxes pass at lockdown commit `0ff245b`.
